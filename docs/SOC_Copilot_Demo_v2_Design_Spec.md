## Part 2: Multi-Scenario Demo

### 2.1 Three Scenarios (The Trifecta)

**Why Three?**
- One scenario = demo
- Three scenarios = product
- Each shows different decision path + different evolution

#### Scenario A: False Positive (Travel)
```
Alert:      John Smith login from Singapore
Context:    User has active travel to Singapore
Decision:   FALSE_POSITIVE_CLOSE (confidence: 92%)
Evolution:  PAT-TRAVEL-001 confidence: 91% â†’ 94%
Soundbite:  "The system knew John was traveling."
```

#### Scenario B: Real Attack (Malware)
```
Alert:      Malware hash detected on SRV-DB-PROD-01
Context:    Critical asset, no travel, production server
Decision:   ESCALATE_INCIDENT (confidence: 97%)
Evolution:  New pattern PAT-MALWARE-PROD-001 created
Soundbite:  "Critical assets escalate. Always."
```

#### Scenario C: Edge Case (CEO After Hours)
```
Alert:      Chris Johnson (CEO) login at 3 AM from new device
Context:    VIP user, high risk score, no travel record
Decision:   ENRICH_AND_WAIT (confidence: 76%)
Evolution:  Threshold for VIP alerts adjusted
Soundbite:  "Not sure? Get more context. Don't auto-close VIPs."
```
